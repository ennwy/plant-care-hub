# plant-care-hub

сайт для догляду за домашніми рослинами. користувач додає свої рослини, дивиться коли поливати, читає поради інших

## функціонал

- логін / реєстрація
- каталог рослин (тягнеться з perenual api)
- свій "гербарій" — додав рослину, нагадує коли поливати
- стрічка порад від інших користувачів
- коментарі, лайки
- адмінка для модерації коментарів і порад
- темна тема після 20:00

## сторінки

### головна
- лого, меню зверху
- кнопки login / register
- велика картинка + текст про сайт
- привітання змінюється від часу доби (доброго ранку / вечора)
- список з 4 кроків як користуватись сайтом
- маркований список з перевагами
- відео як доглядати за рослинами (youtube embed)
- три останні поради знизу

### каталог рослин
- картки з рослинами (фото, назва, складність догляду)
- фільтр по типу через select
- пошук по назві
- картинка з image map — клік на корінь / стебло / лист веде на сторінку з порадами по тій частині

### детальна рослини
- опис, родина, ареал
- умови (полив, світло, температура, вологість)
- галерея
- кнопка "додати до мене"
- останні поради по цій рослині

### мій гербарій
- мої рослини у вигляді карток
- дата наступного поливу на кожній
- віджет погоди по місту користувача (openweather, з лоадером)
- таблиця по днях тижня — що коли робити
- форма додати рослину:
  - вид (select з api)
  - своя назва (input)
  - дата придбання (input date)
  - локація — підвіконня / балкон / кухня (select)
  - частота поливу (range)
  - нотатки (textarea)
  - фото (input file)
- кнопка "полив зроблено" — оновлює таблицю

### спільнота
- стрічка порад
- автор, фото, текст, лайк, коментар
- форма додавання своєї поради
- пагінація — підвантажує наступні через ajax
- коментарі теж підвантажуються порціями

### логін / реєстрація
- логін
- пароль
- підтвердження (для реєстрації)
- аватар (опційно)
- місто (для погоди)
- валідація на blur
- повідомлення про помилки без перезавантаження
- паролі в bcrypt

### профіль
- аватар, логін, дата реєстрації, місто
- скільки рослин, скільки поливів, скільки порад
- кнопка вийти

### адмінка
- доступна тільки для ролі admin
- таблиця всіх порад, кнопки approve / reject
- таблиця користувачів, можна заблокувати
- непідтверджені поради не видно іншим

## навігація

- головна
- каталог
- мій гербарій (тільки для залогінених)
- спільнота
- профіль (тільки для залогінених)
- адмінка (тільки для адміна)
- логін / реєстрація (тільки для незалогінених)

## динаміка

- привітання залежить від часу доби
- темна тема після 20:00 через cookie
- рослини підвантажуються з perenual через ajax
- погода з openweather
- стрічка порад оновлюється без reload
- коментарі підвантажуються порціями
- лайки оновлюються при кліку
- адмін-кнопки тільки для адмінів
- повідомлення про помилки і успіх в окремому місці

## типи контенту

- текст
- таблиці (графік поливу, адмінка)
- зображення (рослини, аватари)
- image map на схемі рослини
- відео (youtube embed)
- форми
- кнопки, select, range

## стек

- фронт: react, без bootstrap і tailwind — свій css (вимоги лаби 2)
- бек: node + express
- зберігання: json-файли (`users.json`, `plants.json`, `tips.json`, `comments.json`) — лаба 5 дозволяє
- авторизація через express-session + bcrypt
- зовнішні api:
  - perenual — рослини
  - openweather — погода
- swagger для документації api (бонусні бали на лабі 6)

## по лабах

| лабораторна | задача                                               |
|-------------|------------------------------------------------------|
| 1           | цей опис + ескіз головної + схема навігації          |

| 3           | js валідація форм (blur + submit)                    |
| 4           | переробляю на react, додаю погоду і perenual         |
| 5           | бек на express, реєстрація, json файли, ssr частково |
| 6           | rest api, роутинг в реакті, swagger                  |
